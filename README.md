# Монитор сетевых соединений

Это простая Java-программа, которая:  
- Смотрит, какие порты заняты на вашем компьютере,  
- Показывает, какие программы их используют,  
- Проверяет, свободен ли конкретный порт (например, 8080).  

Работает на **Windows** и **Linux**.

---

## Как запустить

### 1. Убедитесь, что у вас установлено:  
- **Java 17** (или новее)  
- **Git** (для скачивания проекта)  

---

### 2. Скачайте проект  
```
git clone https://github.com/lokensword/netstat-monitor
cd lab3  
```

---

### 3. Создайте файл настроек  

В корне проекта создайте файл **`app.properties`** со следующим содержимым:  

``` 
monitor.port.check=8080  
output.mode=VERBOSE  
```  

> **Важно**:  
> - `monitor.port.check` — порт, который программа будет проверять.  
> - `output.mode` — можно писать `VERBOSE` (всё показывать) или `SUMMARY` (только если порт занят).

---

### 4. Соберите и запустите  

В терминале (в папке проекта) выполните:  

**На Windows:**  
``` 
gradlew.bat run  
```  

**На Linux/macOS:**  
```
./gradlew run  
```

---

### 5. Что вы увидите  

Пример вывода в режиме `VERBOSE`:  

```
--- Listening Ports ---  
PID: 1234, Process: mysqld, Port: 3306  

--- External Connections ---  
PID: 5678, Process: chrome, Foreign: 104.18.3.10:443  

--- Port 8080 Check ---  
Port 8080 is not in use.  
```

Если поменять `output.mode=SUMMARY` и порт **занят**, вы увидите только:  

```
--- Port 8080 Check ---  
Port 8080 is in use by PID: 9999 (java)  
```

Если порт свободен — **ничего не выведется** (это нормально!).

---

## Как проверить, что всё работает  

### Запустите тесты:  
```
./gradlew test  
```  
Если написано `BUILD SUCCESSFUL` — всё в порядке.

### Соберите JAR-файл:  
```
./gradlew build  
```  
Готовый файл будет в папке `build/libs/`.

---

## Важно  

- Не коммитьте `app.properties` в Git — он уже исключён через `.gitignore`.  
- Программа **не требует прав администратора**, но на Windows может не видеть имена системных процессов (например, `System`), и покажет `N/A` — это нормально.
