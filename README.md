# Монитор сетевых соединений

Приложение для анализа активных сетевых соединений через утилиту `netstat`.  
Поддерживает Windows и Linux. Автоматически определяет ОС, выполняет соответствующую команду, парсит вывод и генерирует отчёт.

## Технологии
   - Java 17
   - Gradle (сборка и управление зависимостями)
   - JUnit 5 (тестирование)

## Пример работы

### Вход (автоматически получается из `netstat`)

    Proto  Local Address      Foreign Address    State       PID
    TCP    0.0.0.0:135        0.0.0.0:0          LISTENING   1216
    TCP    192.168.0.141:49665 20.189.173.15:443  ESTABLISHED 784

### Выход (stdout)

    --- Listening Ports ---
    PID: 1216, Process: svchost, Port: 135
    
    --- External Connections ---
    PID: 784, Process: firefox, Foreign: 20.189.173.15:443
    
    --- Port 135 Check ---
    Port 135 is not in use.


## Сборка проекта

Убедитесь, что установлены:
    
    - Java 17
    - JDK 17
    - Git
    - Gradle
    
Откройте консоль и скачайте репозиторий. Это можно сделать с помощью команды:

    git clone https://github.com/lokensword/netstat-monitor

Затем перейдите в каталог проекта. Он будет иметь имя репозитория. Выполните переход с помощью команды cd:
    
    cd netstat-monitor

Теперь вы находитесь в корне проекта.

## Конфигурация

Создайте файл app.properties в корне проекта. Вы можете сделать это с помощью следующей команды:

    touch app.properties 
    
Далее откройте файл и запишите следующие параметры (укажите нужный вам порт, а также параметр видимости):

    ### Порт для проверки занятости
    monitor.port.check=135

    ### Режим вывода (VERBOSE (всё) или SUMMARY (только проблемы))
    output.mode=VERBOSE

### Учтите: app.properties обязателен для запуска. Без него приложение завершится с ошибкой.

## Сборка проекта

Выполните в корне проекта:
    
    gradlew build

## Запуск приложения

Выполните в корне проекта:

    gradlew run
    
## Запуск тестов

Выполните в корне проекта:
    
    gradlew test

Приложение выведет три раздела или только проверит занятость порта (в зависимости от установленного параметра в app.properties):

Слушающие порты (LISTEN/LISTENING)
Установленные внешние соединения (ESTABLISHED)
Результат проверки указанного порта
